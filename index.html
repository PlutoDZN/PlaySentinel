Features

Session-based risk scoring

Many harmful conversations escalate gradually. PlaySentinel keeps a rolling session context per (user_id → target_id) and computes cumulative risk.
Explainability
Returns categories + matched patterns to support moderator decisions.

Normalization

Detects common obfuscations and aliases like dc/discord, sc/snap/sn4p, etc.
Rate-limited API
Simple API key auth + in-memory throttling for MVP deployments.

API example

POST /v1/analyze
curl -X POST http://localhost:8000/v1/analyze \ -H "X-API-Key: test123" \ -H "Content-Type: application/json" \ -d '{"message":"do you have discord?","user_id":"u1","target_id":"u2"}'
Tip: use consistent user_id and target_id to see session escalation.

Response you can act on

  • score: risk for this message
  • conversation_risk: cumulative session risk
  • risk_level: SAFE / LOW / MEDIUM / HIGH / CRITICAL
  • stage: conversation stage
  • categories: per-category scores
  • matched: matched patterns for explainability
Actions
Block message • Shadowban sender • Alert moderators • Lock chat • Create incident report

FAQ

Does PlaySentinel store chat messages?
Logging is optional. You can store message hashes (no plaintext storage) via SAFEPLAY_LOG_MESSAGES=0.
Is this production-ready?
It’s an MVP. Production typically adds Redis/DB for sessions, monitoring, and richer datasets/ML models.
How do I integrate it into Unity/Roblox?
Call the API from your backend. Use the response fields to trigger moderation actions in your game or platform.
