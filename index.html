How is the score calculated?
Each message increases the message score based on signals like: age questions, platform switching, secrecy, gifts & trust-building. conversation_risk is the cumulative sum across the session.

Features

Session-based risk scoring

Harmful conversations often escalate slowly. PlaySentinel keeps context per (user_id → target_id) and calculates cumulative risk.
Explainability
Categories + matched patterns help moderators decide.

Normalization

Detects obfuscation/aliases like dc/discord, sc/snap/sn4p, etc.
Rate-limited API
API key auth + throttling for MVP deployments.

How it works

Send a message to /v1/analyze. PlaySentinel normalizes text, matches patterns, updates the session, and returns score, stage, and reasons.
Example output (short)
risk_level + stage + reasons → usable directly for actions.

API example

POST /v1/analyze
curl -X POST http://localhost:8000/v1/analyze \ -H "X-API-Key: test123" \ -H "Content-Type: application/json" \ -d '{"message":"do you have discord?","user_id":"u1","target_id":"u2"}'
Tip: use consistent user_id and target_id to see session escalation.

Response you can act on

These fields can directly trigger moderation actions:
  • score: risk of this message
  • conversation_risk: cumulative risk (session)
  • risk_level: SAFE / LOW / MEDIUM / HIGH / CRITICAL
  • stage: conversation stage
  • categories: per-category scores
  • matched: matched patterns (explainability)
  • reasons: signals (e.g., platform_switch_detected)
Actions (examples)
Block message • Hide message • Alert moderators • Lock chat • Create incident report

FAQ

Does PlaySentinel store chat messages?
Logging is optional. You can store hashes instead via SAFEPLAY_LOG_MESSAGES=0.
Is this production-ready?
It’s an MVP. Production typically needs Redis/DB sessions, monitoring, and better datasets/ML.
Unity / Roblox integration?
Best approach: call the API from your backend and trigger moderation actions based on the response.
